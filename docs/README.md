# Documentation map

- `design.md` — target architecture and abstraction boundaries.
- `status.md` — current implementation and verification state.
- `runbook.md` — setup, validation, planning, deployment, and CLI commands.
- `decisions/` — architecture decision records.
- `../TASKS.md` — ordered backlog and acceptance criteria.
- `../AGENTS.md` — operating instructions for coding agents.

Update `status.md` at the end of any material implementation or deployment turn.
Add an ADR when changing a decision that future work would otherwise reasonably
assume is stable.
